1. FONTE DE OBTENÇÃO DOS DADOS

Os dados foram extraídos da API pública da CoinGecko (https://www.coingecko.com/pt/api). Esta é uma das fontes mais utilizadas no mercado financeiro para consulta de dados em tempo real sobre ativos digitais (criptomoedas). Os dados são fornecidos estruturados no formato JSON.

2. CONTEXTO DE APLICAÇÃO DA BASE DE DADOS

A base de dados foi aplicada no contexto de Inteligência de Mercado e Monitoramento Financeiro. O objetivo é criar um repositório histórico de preços de ativos como Bitcoin e Ethereum. Essa aplicação é fundamental para analistas que precisam identificar tendências de mercado, volatilidade de preços e realizar a automação de relatórios para tomada de decisão em investimentos.

4. PROCESSO DE IMPORTAÇÃO PARA O POSTGRESQL

O processo de importação foi realizado através de uma rotina automatizada de ETL (Extração, Transformação e Carga) utilizando a linguagem Python. O procedimento detalhado foi:
Ferramentas: Python 3.14 e banco de dados PostgreSQL 18 (gerenciado pelo pgAdmin 4).
Extração: Utilizamos a biblioteca 'requests' para realizar uma requisição HTTP do tipo GET na API, capturando os dados brutos em JSON.
Transformação: O script Python tratou os dados, isolando os valores decimais de preço e formatando os nomes das moedas.
Conexão: Utilizamos o driver 'psycopg2' para estabelecer a comunicação entre o script e o servidor local do banco de dados na porta 5433.
Carga: Através de um cursor SQL, o script executou comandos do tipo 'INSERT' na tabela 'consulta_moedas', enviando os dados tratados de forma persistente para o banco.
